一、spring-test 包的作用及常用注解

1️⃣ spring-test 是干什么的？

spring-test 的作用：

👉 集成 Spring 测试环境
👉 在单元测试时自动创建 IoC 容器
👉 让我们可以直接使用依赖注入

没有 spring-test：

  • 需要手动 new 容器
  • 手动获取 Bean

有了 spring-test：

  • 自动创建容器
  • 自动注入 Bean
  • 专门为测试优化

本质：

在 JUnit 执行前，帮我们把 Spring 容器启动好。

2️⃣ 常用注解

@SpringJUnitConfig

或者

作用：

  • 启动 Spring 容器
  • 加载配置
  • 支持自动注入

3️⃣ 依赖范围

说明：

spring-test 只在测试阶段使用。

二、JdbcTemplate 常用三个方法

JdbcTemplate 是 Spring 对 JDBC 的封装，核心是：

👉 简化 JDBC 操作
👉 自动处理连接、异常、资源释放

1️⃣ update()

用于：

DML 操作（insert / update / delete）

返回值：

  • 影响的行数

2️⃣ queryForObject()

用于：

查询单个结果

说明：

  • 返回单个对象
  • 结果必须唯一

3️⃣ query()

用于：

查询多个对象

说明：

  • 返回 List 集合
  • 使用 RowMapper 映射对象

三、什么是代理技术？

1️⃣ 为什么需要代理？

问题：

👉 非核心代码冗余问题

例如：

  • 日志
  • 事务
  • 权限控制
  • 异常处理

如果直接写在业务类中：

  • 代码混乱
  • 重复严重
  • 不利于维护

2️⃣ 代理技术是什么？

代理技术是一种：

👉 间接调用方法的技术
👉 将非核心代码提取到代理类
👉 核心代码保留在目标类

执行流程：

客户端 → 代理类 → 目标类

实现了：

  • 功能增强
  • 代码解耦
  • 核心代码纯净

四、静态代理 vs 动态代理

1️⃣ 静态代理

特点：

  • 需要手动写代理类
  • 编译前就存在
  • 目标类变化，代理类也要改

缺点：

👉 麻烦
👉 类数量爆炸

2️⃣ 动态代理

特点：

  • 不需要手动写代理类
  • 运行时生成代理对象
  • 自动增强方法

优点：

👉 简洁
👉 灵活

五、JDK 动态代理 vs CGLIB

1️⃣ JDK 动态代理

特点：

  • 要求目标类必须有接口
  • 代理类是接口的实现类
  • 代理类和目标类是“兄弟关系”

本质：

👉 基于接口

2️⃣ CGLIB 动态代理

特点：

  • 不需要接口
  • 代理类是目标类的子类
  • 通过继承实现增强

本质：

👉 基于继承

3️⃣ 使用注意

一旦使用 AOP：

👉 有接口必须使用接口类型接收

推荐：

👉 永远用接口接收

六、AOP 配置相关注解

1️⃣ 核心注解

  • @Aspect → 声明切面类
  • @Before → 前置通知
  • @AfterReturning → 返回通知
  • @AfterThrowing → 异常通知
  • @After → 后置通知
  • @Around → 环绕通知
  • @Pointcut → 抽取切点表达式

2️⃣ 启用 AOP

3️⃣ 多切面执行顺序

值越小优先级越高。

七、切点表达式语法

切点表达式格式：
